Проблемы CSS

  1. CSS сложно понять без контекста (HTML)
  2. Правила могут повторяться и переопределять друг друга
  3. Часть свойств наследуются
  4. Почти нет группировки
  5. Количество CSS всегда растет

Решения проблем

  1. Соглашения о написании кода: @mdo, cssguidelin.es, google, тысячи их.
  2. Методологии именования селекторов: OOCSS, SMACSS, БЭМ
  3. Грамотная файловая организация
  4. Грамотное комментирование кода

БЭМ

Три типа селекторов: Блок. Элемент (блока). Модификатор

Полный стек и только метод именования селекторов.

Модификация Николаса Галлахера.

БЭМ: Почему он лучший

Даёт абсолютно независимые блоки

  1. Предсказуемость работы кода
  2. Безопасная модификация блоков
  3. Самодокументируемость CSS — код понятен без контекста
  4. Реиспользуемость (в т.ч. кросспроектная)

БЭМ: Три типа классов

Блок — абсолютно самостоятельная сущность. Из блоков собирается страница.

Примеры: запись в блоге, кнопка, карточка товара в рубрике каталога.

Элемент — часть блока, абсолютно несамостоятельная сущность, не может встречаться вне своего блока

Примеры: «шапка» записи в блоге, название товара в карточке товара.

Модификатор — модификатор внешнего вида блока или элемента.

Пример: модификатор фонового цвета кнопки.

БЭМ: Блок

Имя класса блока должно отвечать на вопрос «что это такое?»

Имя класса блока НЕ должно отвечать на вопрос «как это выглядит?».

Блоки могут быть вложены друг в друга.

		  .btn {
		    color: blue;
		    border-radius: 5px;
		    background: #eee;
		  }

БЭМ: Элемент

Имя класса элемента должно отвечать на вопрос «что это такое?»

Имя класса элемента НЕ должно отвечать на вопрос «как это выглядит?».

Составление класса элемента: имя-блока__имя-элемента

		  .main-nav__list {}
		  .main-nav__list-item {}
		  .main-nav__link {}
		  .blog-post__header {}

БЭМ: Модификатор

Имя класса-модификатора должно говорить о сути модификации (ключ-значение или только значение)

Класс-модификатор задаёт только модификацию и не должен встречаться в HTML без того класса, который он модифицирует.

У одного блока допустимо несколько модификаторов.

БЭМ: Модификатор

Составление класса модификатора:

БЭМ Яндекс: имя-блока_имя-модификатора

Модификация Николаса Галахера: имя-блока--имя-модификатора

		  .btn--large {}
		  .blog-post--border-blue {}

БЭМ: Проблемка...

Как отличить элемент блока от вложенного блока?

ЭТО может с высокой вероятностью потребоваться вне родителя?
Да, может — это вложенный блок.
Нет, только внутри родителя — это элемент

БЭМ: Правила

  1. Селектор — только класс
  2. Избегаем вложенных селекторов
  3. Отказываемся от !important
  4. Минимум глобальных стилей (типографика)
  5. Избегаем тегов в селекторах
  6. HTML: минимум тегов, не покрытых классами
  7. Избегаем классов-не-блоков

БЭМ: Уровни переопределения

Конечная реализация блока может быть разделена по разным уровням переопределения.

Каждый последующий уровень добавляет или перекрывает исходную реализацию блока.
